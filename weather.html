Weather App

Features

    /
  • Accesses the mongodb database, via a springboot backend endpoint to check credentials.
  • Allows the user to search towns via search option.
      Converts town names into zipcodes
      If a zipcode/town name is invalid, will catch the error and notify the user on the town page.
  • Access registration page, to make a new unique account.
      • /register
  • Requires unique username and email.
  • Requires all data be filled out before attempting to register.
      • /town?searched_town=
  • Allows user to search for towns on dashboard bar.
  • Queries data for searched town, and displays the weekly outlook.
  • Shows active weather alerts for the town.
  • Home navigation button:
      If logged in, it will take you to your home page.
      Otherwise, it will take you back to the root/login page.
      • /home
  • Queries weather data for home location (modifyable in edit_profile page)
  • Allows you to add or remove a list of towns from your favorites list (searchable by their ip adress)
  • Displays username, along with avatar (both modifyable in edit_profile page)
      • /edit_profile
  • Allows the user to log out.
  • Allows the user to change the following things:
      Username (must be unique)
      Password
      Home Town
      Avatar

      • Technologies

        Springboot Backend
        Communicates with mongodb, and performs CRUD operations on user data.
        Flask
        Uses endpoints from the Springboot backend.

        Communicates with weather api and accesses the weather data, as well as performing checks on town names and zip codes.

        Responisble for route logic, and properly gathering data for displaying on pages.
